## 🛠️ What I'm building

### Compozy — the agent harness

The core thesis: AI agents are powerful but unreliable on their own. They need a **harness** — orchestration, durable state, peer discovery, and quality gates — to turn one-shot generations into shippable software.

| Project | What it does |
| --- | --- |
| **[compozy](https://github.com/compozy/compozy)** | CLI that drives the full lifecycle of AI-assisted dev — PRDs → specs → tasks → concurrent execution → code review remediation. |
| **[agh](https://github.com/compozy/agh)** | An open workplace for AI agents. Runs Claude Code, Codex, OpenClaw and friends as durable, inspectable sessions with peer-to-peer capability sharing. |
| **[skeeper](https://github.com/compozy/skeeper)** | Sidecar Git repo for spec artifacts. Keeps PRDs, ADRs, and AI plans versioned and diffable — separate from the main branch noise. |
| **[codex-loop](https://github.com/compozy/codex-loop)** | Release-grade QA loop for Codex CLI. Iterates until duration / rounds / goal targets are met — not just on the first plausible answer. |
